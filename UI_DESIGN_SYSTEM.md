# UI DESIGN SYSTEM – Library Management System
*(Visual Style Reference – Non-functional)*

## 1. Design Philosophy

The UI design follows a modern, clean, and minimalistic style inspired by contemporary BookStore eCommerce interfaces.
The reference is used strictly for visual appearance, layout structure, and typography consistency, not for functional workflows.

Design goals:
- High readability and visual clarity
- Soft, friendly, and professional appearance
- Card-based and grid-oriented layout
- Clear hierarchy for data-heavy interfaces
- Suitable for dashboards and management systems

## 4. Layout Principles

### 4.1 Overall Layout
- Centered content with max-width
- Generous white space
- Consistent padding and margin system
- Responsive grid-based structure

### 4.2 Navigation
- Top navigation bar
- Clear separation between navigation and content
- Subtle background contrast

### 4.3 Cards
- Rounded corners
- Soft shadow
- White background on light page
- Used for:
  - List items
  - Summary blocks
  - Information containers

---

## 5. Component Style Guidelines

### Buttons
- Rounded corners
- Clear hover and active states
- Primary / Secondary / Status variants

### Tables
- Light header background
- Thin borders
- Clear row separation
- Emphasis on readability

### Forms
- Minimal input borders
- Clear focus state
- Inline validation feedback

---

## 6. Responsiveness

- Desktop-first design
- Adaptive grid for tablets
- Vertical stacking for mobile
- Touch-friendly spacing on small screens

---

## 7. Scope Clarification

This design system defines:
- Visual style
- Color usage
- Typography
- Layout principles
- Component appearance

It does NOT define:
- Page navigation structure
- Functional workflows
- Business logic
- User roles or permissions

All functional behavior is implemented separately based on the Library Management System requirements.
